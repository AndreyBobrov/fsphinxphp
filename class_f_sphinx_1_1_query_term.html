Detailed Description

A class that represents a single query term for a Sphinx query.

Creates a single query term used internally by the MultiFieldQuery object. A raw query term is of the form "@genre drama" where "genre" is the field to filter by and "drama" is the filtering term. Numerical terms may also be used if the FSphinx client is configured to filter by multi-valued attribute IDs.

Query terms may be created from an FSphinx result array or a string representation.

Parameters:
string$statusWhether a query term is active. '+' for on, '-' for off.
string$fieldField to filter by.
string$termTerm to filter by.
array$user_sph_mapMapping between user fields and actual Sphinx fields.
array$user_attr_mapMapping between user fields and Sphinx multi-value attributes.
